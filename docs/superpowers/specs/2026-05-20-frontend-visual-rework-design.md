# Дизайн: візуальна поліровка фронтенду (хедер · hero · футер · кнопки · форми)

- **Дата:** 2026-05-20
- **Статус:** узгоджено, готово до плану реалізації
- **Скоуп:** `lyceum-157-frontend`
- **Брейншторм-макети:** `.superpowers/brainstorm/724-1779285663/content/` (header-at-top, button-redesign, heroes-v2, nav-hover-v2, footer-rework тощо)

## 1. Контекст і проблема

Публічна частина магазину має сильну візуальну мову (паперово-архівна естетика
MASTER-spec / `DESIGN.md`), але страждає від **неохайного виконання**: статичний хедер,
гігантські непослідовні пробіли між секціями, перевантажений 4-блоковий футер,
кнопки з поламаним hover-контрастом, форми входу без композиції.

Мета — **підняти якість виконання**, зберігши насичений архівний стиль. Це не
мінімалізм і не редизайн концепції: марки, фото, печатки, іконки, тепле тло —
лишаються. Виправляємо те, як це зібрано.

## 2. Цілі / поза скоупом

**Цілі:** хедер (стани при скролі), навігація, вертикальний ритм секцій, hero на
ключових сторінках, футер, первинна/вторинна кнопки, екрани `login` + `register`,
поліровка форм login/cart.

**Поза скоупом:** кабінети (`student/admin/parent`), checkout-флоу, бекенд-контракт,
нові сторінки. Зміни спільних компонентів (`Section`, `EditorialDivider`, кнопка)
впливають на весь сайт — це навмисно, для консистентності.

## 3. Принципи

1. Зберігати насичену архівну естетику; «AI-слоп» = неохайність, лікуємо її, а не декор.
2. Тільки токени (`tokens.css` / Tailwind theme) — жодних hardcoded кольорів/розмірів.
3. Не ламати FSD-межі (ESLint-boundaries), імпорти через `@/`.
4. `stamp-drop` не замінювати на fade-in; будь-який новий рух поважає `prefers-reduced-motion`.

## 4. Хедер (`widgets/header`)

Два стани замість постійного `bg-bg/90 backdrop-blur-md`:

- **rest** (`scrollY ≤ 8`): фон **прозорий**, без `border-b`, без тіні; топ-бар видимий.
  Хедер «зливається» зі сторінкою/hero. Лінки та іконки фарбуються за **тоном hero**.
- **floating** (`scrollY > 8`): фон — матове скло (`--header-glass-bg`, ~`rgba(250,250,247,.72)`)
  + `backdrop-blur` (~12px) + hairline `border-b border-line` + `shadow-paper`;
  топ-бар згорнутий (`h-0`); висота хедера зменшена (~64–72px). Лінки завжди `ink`.
- Перехід між станами — `duration-d3` `ease-paper`.

**Тон hero.** Вводимо легкий механізм `HeaderTone` (React-контекст або
`data`-атрибут): сторінки з темним фото-hero — **Каталог і Автори** — оголошують
`tone="dark"` → у стані rest лінки/іконки кремові (`bg-warm`/білий). **Головна,
Про проєкт, Колекції, Контакти** — `tone="light"` (дефолт, тепле/світле тло) →
лінки `ink`. У стані floating тон ігнорується (завжди світле скло).

Мобільний drawer не змінюється.

## 5. Навігація (`widgets/header/nav.tsx`)

Прибрати підкреслення-grow. Замість нього — **один індикатор-рамка**, що ковзає:

- Абсолютний елемент-рамка: `1.6px solid burgundy`, заокруглений, **без заливки**.
- JS міряє `getBoundingClientRect` лінки під курсором, виставляє `left`/`width`.
- Перехід `left`/`width` ~`520ms` `ease-paper` (повільний, неквапливий глайд).
- Спокій — рамка на активній лінці; `mouseleave` навігації → повернення на активну.
- `prefers-reduced-motion` → індикатор переміщується без transition.
- Типографіка лінок — токенні класи (`label`/`small`), не hardcoded `text-[13px]`.
- На темному хедері рамка/текст — кремові; на світлому — бордові.

## 6. Вертикальний ритм секцій

Прибрати непослідовні пробіли («великі й негарні»):

- **Верх сторінки.** Зараз `main` має `pt-[124px]` + `EditorialPageShell` → `Section pad="default"`
  додає ще `py-120` → ~244px порожнечі. Верхній відступ контент-секцій зменшити до
  `~32px` (моб) / `~48px` (десктоп). Нижній — лишити просторим (`~64/96px`).
- **Між блоками.** Єдиний ритм `40px` (моб) / `64px` (десктоп) замість `Stack gap={6}` (32px)
  + ad-hoc. `EditorialPageShell` віддає цей ритм (responsive gap або `space-y-*`).
- **Розділювачі.** `EditorialDivider` як сепаратор блоків не додає власних
  вертикальних відступів (варіанти `marks/number/star` зараз мають `py-4`) — ритм
  64/40 уже включає розділювач.
- **`home-screen.tsx`.** Прибрати ad-hoc `py-12` / `py-8` / `space-y-12` по секціях,
  звести до спільного ритму.
- Hero на сторінках з hero — повноширинний, лежить одразу під хедером, перед `EditorialPageShell`.

## 7. Hero — окремий концепт на сторінку

Спільна мова (Fraunces-заголовки, мітки ▌, печатки, фото, папір, photo-print тіні),
**різна композиція**. Зображення — реальні з `public/images/**` через `ImageSlot`
(синк `scan-images`).

| Сторінка | Hero | Wow |
|---|---|---|
| Головна | Masthead (▌-мітка + `mega` курсив «Майстерня 157» + лід + печатка) злитий з колажем 3 розкиданих полароїдів. Прибрати розділювач між masthead і колажем — це один hero-блок. | Паралакс/дрейф полароїдів, `stamp-drop` печатки |
| Каталог | Повноширинне документальне фото (`catalog/hero/banner.webp`) з градієнт-скримом зліва, заголовок `display`/`h1` + ▌-мітка поверх, печатка, музейна підпис-картка знизу. | Паралакс фону, `stamp-drop` |
| Автори | Щільна стіна Ч/Б портретів-облич (grayscale, **без чорних ліній** — стик впритул), бордовий блок із назвою поверх по центру. | Хвильове проявлення портретів |
| Про проєкт | Тихий розворот-маніфест: велике Ч/Б фото на одній половині + текст-заява (▌-мітка + `h1` + лід) на іншій. | Проявлення тексту рядками |
| Колекції, Контакти | **Без великого hero** — простий охайний заголовок-смуга (▌-мітка + Fraunces-назва, компактно, тепле тло). | — |

- Паралакс — лише Головна + Каталог; усе проявлення/паралакс під `prefers-reduced-motion`
  вимикається.
- Потрібен хук скрол-паралаксу в `shared/hooks` (поряд із наявним `use-intersection.ts`).
- Хедер над темним hero отримує `tone="dark"` (див. §4).

## 8. Футер (`widgets/footer/footer.tsx`)

Лишаємо **зворотну листівку + смужку копірайту**, прибираємо **лицьову листівку +
перфорацію-шов**.

- **Видалити:** блок лицьової листівки (SVG-ескіз будівлі + печатка EST 1957);
  перфорацію-шов із 32 «дірок».
- **Лишити (стиль зберігається):** зворотну листівку з пунктирною рамкою — лист,
  підпис директорки, поштова марка (`PostageStamp`), воскові печатки (`Stamp`),
  адреси двох корпусів, соц-іконки; теплий фон; водяний знак «1 5 7».
- **Поліровка:** розкривний toggle «обери розділ» прибрати — навігаційні лінки
  завжди видимі; смужку копірайту звести до одного рядка (прибрати дубльований
  «★ made by…»); прибрати easter-egg «лизни марку» з конюшиною; зменшити
  `mt-24` зверху футера до `~mt-12/16`; водяний знак «157» перепозиціонувати під
  одноблочний футер.

## 9. Кнопки (`shared/ui/pill-button`)

Повний редизайн форми кнопки (пігулку прибираємо). **Свідоме, узгоджене з
користувачем відхилення від `DESIGN.md`** (там кнопка `rounded.full`): пігулка
сприймається як «стандартний слоп»; гострі кути сигналять «надруковано».

- **Первинна — «друкований блок»:** `rounded-none`, заливка `burgundy`, текст —
  `label` (uppercase, трекінг), білий; жорстка зміщена тінь `--sh-press`
  (~`6px 6px 0 bg-noir`); опційний знак `→`.
  - **hover:** `translate(5px,5px)` (вдавлюється в тінь) + тінь стискається до `2px 2px`
    + заливка `burgundy → green` (фірмовий чорнильний свап з `DESIGN.md` зберігається).
  - **active:** повністю втиснута.
- **Вторинна — «чорнильний контур»:** прозора, `2px solid burgundy`, `rounded-none`,
  тонка зміщена тінь; hover — заливка `green`, текст білий.
- Компонент лишається в `shared/ui` (назву можна змінити з `PillButton` на `Button`).
- **Прибрати override-класи на всіх викликах** (`login-form`, `cart-screen` empty-state
  і checkout): жодних `hover:bg-burgundy-soft` / `text-bg-warm` / `hover:bg-burgundy/90` —
  лише `variant` + layout-класи (`w-full`, `mt-*`).
- Степер кількості в кошику — вирівняти й узгодити з новою мовою (гострі кути).
- Новий токен у `tokens.css`: `--sh-press` (і пресований стан).

## 10. Екрани `login` + `register`

Обидва — **двоколонкові, дзеркальні** (на мобільному стек, фото зверху або сховане):

- **Реєстрація:** форма **зліва**, фото **справа**. Структура форми лишається
  («нормальна»), лише стиль уніфікується + нова кнопка.
- **Логін:** фото **зліва**, форма **справа** — дзеркало реєстрації. Заголовок «Вхід»
  зменшити з `mega` (до 200px) до `display`/`h1`. Декоративну печатку прибрати від
  переповнення.
- Фото-колонка — зображення з `public/images/auth/**` (+ можлива печатка/підпис-картка).
- Стиль полів єдиний для обох; «бібліотечна» підкреслена форма інпутів зберігається,
  чіткий focus-стан (бордове підкреслення). Замість центрованого `EditorialPageShell`
  — окремий двоколонковий auth-layout.

## 11. Поліровка форм login / cart

- Кнопки login (submit) і cart (checkout, «До каталогу») → нова первинна кнопка,
  без поламаних override (див. §9).
- Інпути login/register — консистентні, видимий focus.
- `useAppForm()` обгортка не змінюється.

## 12. Токени (`_app/styles/tokens.css`)

Додати:

- `--header-glass-bg: rgba(250, 250, 247, 0.72)` — фон матового скла хедера.
- `--sh-press: 6px 6px 0 var(--c-bg-noir)` — зміщена тінь кнопки (+ пресований `2px 2px`).
- За потреби — `--blur-glass: 12px`.

Tailwind theme розширити відповідними утилітами (boxShadow/backgroundColor), щоб у
компонентах лишалися тільки класи.

## 13. Рух і доступність

- Усе нове (паралакс, проявлення hero, глайд індикатора навігації) — під
  `prefers-reduced-motion` вимикається або стає миттєвим. `tokens.css` уже занулює
  тривалості — нові ефекти теж мають це поважати.
- `stamp-drop` лишається пружинним; fade-in для печаток заборонено.
- Контраст тексту на всіх станах хедера/кнопок — ≥ 4.5:1 (WCAG).

## 16. Відхилення від `DESIGN.md`

- **Кнопка:** `rounded.full` (пігулка) → `rounded.none` «друкований блок». Свідоме,
  узгоджене з користувачем рішення; фірмовий hover `burgundy→green` і токени збережено.
- Гласморфізм хедера у стані floating — `DESIGN.md` його прямо дозволяє для
  прилипаючого хедера.
- Решта рішень — у межах spec (60-30-10, типографіка, печатки, папір).
