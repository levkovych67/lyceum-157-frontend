# РОЗДІЛ I. ОБ'ЄДНУЮЧА КОНЦЕПЦІЯ

## 1.1 Велика метафора

> **Сайт — це 47-й випуск щорічного архіву Майстерні 157.**
> Архів виходить з 1957 року, друкується вручну на теплому молочному папері, ілюструється чорно-білими фото з духом старих фотонарисів, і кожен випуск присвячений учнівським роботам поточного року.

Усе на сайті — **частина видання:**

| Сторінка | Що це у форматі архівного випуску |
|---|---|
| Головна | Обкладинка + перші 20 сторінок (вступ, фотонарис, головні автори) |
| Каталог | Покажчик / contact sheet усіх робіт випуску |
| Товар | Окрема стаття про твір |
| Автор | Профайл-feature |
| Колекції | Тематичні спецвипуски всередині архіву |
| Кошик | Список «забрати додому» |
| Чекаут | Бланк-замовлення (паперова форма) |
| Дякуємо | Поштова листівка-підтвердження |
| Про проєкт | Редакторська колонка |
| Контакти | Колофон / задня обкладинка |
| Вхід | Бібліотечна картка читача |
| Кабінет | Персональний архівний кутик |

**Чому це працює:** одна метафора → одна мова візуалів → жодних style jumps. Кожен новий патерн перевіряється питанням: *«Чи знайшов би я це в архівному виданні 1957–2026?»*

## 1.2 Три стовпи айдентики

**1. ПАПІР.** Усе має tactile-якість. Текстура. Тіні фотопаперу. Молочний фон. Жовті стікери. Карбоновий чорний. Вітальні листівки. Видимі сліди фізичного процесу — нерівні штампи, кутики, нахили.

**2. ТИПОГРАФІКА.** Це видавничий продукт. Заголовки великі, італійські, з opsz-axis. Тексти спокійні. Рукописи на полях. Editorial labels (▌). Letter-spacing у малому 12%. Великі цифри.

**3. ШКОЛА.** Це не «магазин про школу», це **школа**, яка показує свій архів. Печатки 157, шкільний герб у мітках, фото коридорів, рукописи учнів, цитати випускників, адреса корпусів у листівках.

## 1.3 Жорсткі заборони (anti-patterns)

| ✗ ЗАБОРОНЕНО | Чому | ✓ Замість цього |
|---|---|---|
| Symmetric grid 4×N | Виглядає як шаблон Shopify | Asymmetric layout з явним rhythm |
| Photo hero з 2 кнопками | Cliché сучасного e-commerce | Typographic hero з колажем |
| Donut charts, інфографіка | Говорить «корпоративний сайт» | Текст-нарис + фото |
| Gradient bg (purple/pink) | AI-slop сигнатура | Solid molochny + paper noise |
| Glass morphism | Приємно, але говорить «AI» | Прозорий blur тільки на sticky header |
| Generic Lottie animations | Швидко набридає | Текст з paper-rip / page-turn |
| Inter, Space Grotesk, Roboto | Усе AI робить ці шрифти | Fraunces + Manrope + Caveat |
| Smooth purple corner glow | Те саме AI cliché | Stamp у кутку, нахилений |
| Floating chat bubble | Нав'язливо + ламає метафору | Лист у Footer + email у листівці |
| Cookie-banner на півекрану | Ламає магію | Тонка стрічка #1a1612 зверху, dismiss |
| Емодзі в UI | Юний/несерйозний тон | Symbols (★ ▌ → ⊙) як друкарські знаки |

## 1.4 Один сайт — три інтонації

Хоч стиль єдиний, інтонація плавно змінюється від сторінки до сторінки:

- **Editorial-formal** (заголовки, типографіка, рівний тон) — Головна, Каталог, Колекції, Про, Контакти.
- **Editorial-warm** (більше рукописів, цитат, фото-портретів) — Автор, Робота, Дякуємо.
- **Functional-paper** (мінімум декору, але та сама bg/typo, фокус на формі) — Кошик, Чекаут, Вхід, Кабінет.

Інтонація не міняє візуальну мову — просто dimms decoration на functional-сторінках.

## 1.5 ⚠ КРИТИЧНІ ТАКТИЛЬНІ ДЕТАЛІ — НАГОЛОС ДЛЯ STITCH

> **При розробці у Stitch (і взагалі — у фронтенд-реалізації) зверніть особливу увагу на ДВА елементи: текстуру паперу і анімацію печаток. Це ті деталі, які зроблять сайт "тактильним" і виділять його серед будь-яких інших шкільних проєктів.**
>
> Без них сайт перетвориться на ще один WordPress-шаблон. З ними — на видавничий продукт, що його не сплутаєш ні з чим.

### 1.5.1 Текстура паперу (Paper Noise) — must-have

**Що це.** Тонкий шар шуму на повний viewport, що імітує неоднорідність справжнього паперу. Вмикається на body, лежить вище фону, але нижче контенту, не реагує на події.

**Реалізація — три шари (cumulative):**

#### Шар 1 — Базовий шум (на всі сторінки)
```css
body::before {
  content: '';
  position: fixed;
  inset: 0;
  background-image: url('data:image/svg+xml;utf8,<svg xmlns="http://www.w3.org/2000/svg" width="256" height="256"><filter id="n"><feTurbulence type="fractalNoise" baseFrequency="0.85" numOctaves="2" stitchTiles="stitch"/><feColorMatrix values="0 0 0 0 0.43, 0 0 0 0 0.15, 0 0 0 0 0.24, 0 0 0 0.5 0"/></filter><rect width="256" height="256" filter="url(%23n)" opacity="0.5"/></svg>');
  background-size: 256px 256px;
  mix-blend-mode: multiply;
  opacity: 0.04;
  pointer-events: none;
  z-index: 1;
}
```
- `baseFrequency 0.85` — дрібне, як грубий папір.
- `mix-blend-mode: multiply` — дозволяє опустити шум туди, де темно, зберігаючи світлі ділянки.
- Колір noise з тонким бордовим відтінком (matrix values 0.43, 0.15, 0.24) — узгоджено з палітрою, а не нейтрально-сірий.

#### Шар 2 — Зернистість фотографій (тільки на photo блоках)
Усі ч/б photo (Manifesto, Photo Interlude, Author Hero, Featured Collection, About Hero) додатково отримують grain overlay:
```css
.photo-grainy::after {
  content: '';
  position: absolute;
  inset: 0;
  background: url('/textures/film-grain-1024.png') repeat;
  mix-blend-mode: overlay;
  opacity: 0.18;
  pointer-events: none;
}
```
Це робить фото схожим на скан плівки 35mm, а не цифрову зйомку.

#### Шар 3 — Текстура волокна (на postcard back, sticker notes, paper forms)
Поверхні `--c-bg-warm` (#f3ead6) і жовті стікери отримують посилену текстуру (opacity 0.08–0.12) з більшою зернистістю — це робить «теплі папери» окремими від «холодних» поверхонь карток.

**ЯК ПЕРЕВІРИТИ що зроблено правильно:**
- Зменши масштаб браузера до 50% — на блокі однотонного фону повинна бути видна **легка нерівність кольору**. Якщо фон ідеально гладкий — шум не працює.
- Подивись на screenshot — фон не повинен бути піксельно ідентичним між регіонами. Якщо однорідний — opacity занадто низький.
- На retina-екрані шум має лишатися видимим (не зникати від pixel ratio).

**ЧОГО НЕ РОБИТИ:**
- ✗ НЕ використовуй PNG-noise з білим/сірим тоном — він робить сайт «брудним», а не «теплим».
- ✗ НЕ ставити opacity > 0.06 на body — readability страждає.
- ✗ НЕ застосовуй noise через `backdrop-filter` — це ламає performance і не дає того ж відчуття.
- ✗ НЕ забувай про `pointer-events: none` — інакше блокує click-и.

### 1.5.2 Анімація печаток (Stamp Drop) — фірмовий ефект

**Печатки — головний фірмовий артефакт сайту.** Вони з'являються мінімум 5+ разів на головній і 1–2 рази на кожній іншій сторінці. Кожна печатка має **унікальний жест появи** — не просто fade-in.

**Базова анімація — Stamp Drop:**
```css
@keyframes stamp-drop {
  0% {
    transform: translateY(-180px) scale(1.6) rotate(var(--final-rotation, -8deg));
    opacity: 0;
    filter: blur(2px);
  }
  60% {
    transform: translateY(-8px) scale(0.94) rotate(calc(var(--final-rotation, -8deg) * 0.8));
    opacity: 1;
    filter: blur(0);
  }
  72% {
    transform: translateY(2px) scale(1.06) rotate(var(--final-rotation, -8deg));
  }
  85% {
    transform: translateY(0) scale(0.98) rotate(var(--final-rotation, -8deg));
  }
  100% {
    transform: translateY(0) scale(1) rotate(var(--final-rotation, -8deg));
  }
}

.stamp[data-animate="true"] {
  animation: stamp-drop 720ms var(--ease-stamp) both;
}
```

**Що відбувається покроково:**
1. **0–60%** (430ms): печатка падає згори, велика (×1.6), розмита, з обертом до фінального кута.
2. **60–72%** (87ms): «удар» — стискається до 0.94, проходить нижче нульової точки.
3. **72–85%** (94ms): відскок — підстрибує до 1.06.
4. **85–100%** (109ms): затухання осциляції, settle на 1.0.

Це ефект **штампування**: контакт з папером, віддача, мікро-вібрація. На відміну від generic spring-animation, він читається як фізичний жест.

**Додаткові ефекти — для важливих печаток (наприклад "ВРУЧЕНО" на Thank-you, "АРХІВ 157" в Header):**

#### Ink smudge (відбиток чорнила)
При settle печатка лишає легку «тінь чорнила» — duplicate того ж SVG, blur 8px, opacity 0.15, offset +4px по обох осях. З'являється в момент 60% і лишається назавжди. Створює ілюзію свіжого штампу що ще не висох.

#### Stamp shadow trail
Між 0% і 60% за печаткою тягнеться лінія тіні (CSS box-shadow з multiple offsets) що поступово коротшає. Симулює motion blur падаючого об'єкта.

#### Звуковий ефект (опційно, ОДИН раз за сесію)
При першому stamp-drop на сторінці програється `data:audio/wav` короткий «тук» — тембр глухого штампу по дереву. Гучність 0.3. **Тільки якщо** користувач взаємодіяв зі сторінкою (інакше autoplay блокується). Якщо браузер блокує — без помилки, без візуального fallback. Це pure delight, не функціонал.

**Trigger-логіка:**
- На load сторінки — печатки з `data-animate-on="load"` штампуються з staggered delay (0ms, 120ms, 240ms...).
- На scroll-into-view — печатки з `data-animate-on="scroll"` штампуються коли блок входить у viewport (IntersectionObserver, threshold 0.4).
- На hover (тільки великі печатки в Header / Auth) — невелике re-stamp: rotate(±4° від final), 280ms ease.
- На click (печатка-easter-egg в Header) — повна re-animation + відкриття модалки.

**Особливі moments:**
- **Thank-you page**: печатка «ВРУЧЕНО» 200×200 робить **уповільнений** stamp-drop (1200ms), приходить через 400ms після завантаження сторінки. Це ключовий емоційний момент.
- **Game block**: при правильній відповіді штамп «✓ ВІРНО» падає під рандомний кутом ±15° від базового, додає вижатки до конфеті.
- **Stickers (Letters блок)**: НЕ використовують stamp-drop, бо вони НЕ печатки. Вони — паперові стікери. Анімація: tilt-into-place з spring (-rotation final ± 20° → final), 480ms.

**ЯК ПЕРЕВІРИТИ:**
- Зайди на головну вперше — мають бути 3+ печатки що штампуються в перші 2 секунди (в hero, marquee, manifesto).
- Скрол вниз — печатки в gallery wall, CTA, footer штампуються по черзі при появі.
- Hover на печатку в header — має бути микро-rotation.
- Натисни на печатку «АРХІВ 157» в top-bar 5 разів — easter-egg модалка.

**ЧОГО НЕ РОБИТИ:**
- ✗ НЕ роби fade-in замість stamp-drop. Це вбиває ефект.
- ✗ НЕ дублюй stamp-drop одночасно більше ніж на 3 печатках — створює хаос. Завжди staggered.
- ✗ НЕ використовуй `transition` для stamp-drop — тільки `animation` (бо потрібні keyframes 60/72/85%).
- ✗ НЕ забувай respect `prefers-reduced-motion` — у такому режимі печатки просто з'являються opacity 0→1 за 200ms.

### 1.5.3 Чому ці дві деталі важливіші за інші

Сайт без них **залишається дизайнерським**, але стає одним з тисячі. З ними — він стає **впізнаваним з пів-секунди**.

**Тест узнавання:**
> Якщо я зроблю screenshot Hero блоку і покажу 10 дизайнерам поряд з 9 іншими шкільними сайтами — чи виділяться наш через 3 секунди?

Без paper-noise і stamp-animation — навряд. З ними — гарантовано.

**Це інвестиція з найвищим ROI на тактильність.** Усе інше (типографіка, кольори, асиметрія) можуть бути на 90% — і це нормально. Але paper-noise і stamps мають бути на 100%, інакше система не встає.

## 2.1 Кольори — фінальна палітра

```css
:root {

  --c-burgundy-deep:   #4d1a2a;   /* hover, active states */
  --c-burgundy-soft:   #f5e6ea;   /* tinted backgrounds, badges */

  --c-green:           #0c6633;   /* secondary, навігація, success */
  --c-green-deep:      #00662a;   /* deep accent, lines */
  --c-green-soft:      #d8e6dc;   /* soft tinted bg */

  /* === SURFACES === */
  --c-bg:              #fafaf7;   /* MAIN — теплий молочний */
  --c-bg-warm:         #f3ead6;   /* postcard back, warm callout */
  --c-bg-card:         #ffffff;   /* картки, інтерфейсні поверхні */
  --c-bg-yellow:       #fef3c7;   /* стікери, гра, акцент-секції */
  --c-bg-blue:         #dbeafe;   /* стікери варіант */
  --c-bg-noir:         #1a1612;   /* top bar, very bottom, contrast */

  /* === INK === */
  --c-ink:             #1e1e1e;   /* основний текст */
  --c-ink-soft:        #6b6b6b;   /* secondary text */
  --c-ink-fade:        #a8a8a8;   /* tertiary, hints */
  --c-line:            #ececec;   /* розділювачі */
  --c-line-strong:     #c9c0b3;   /* акцент-лінії на warm bg */

  /* === FUNCTIONAL === */
  --c-link:            #1e73be;   /* посилання у плотному тексті */
  --c-stamp:           #6e273d;   /* печатки */
  --c-error:           #b03030;   /* помилки форми */
}
```

**Правило 60-30-10:**
- 60% поверхні — `--c-bg` (молочний)
- 30% акцент-блоки — `--c-burgundy` (бордовий) + рідше `--c-bg-warm`/`--c-bg-yellow`
- 10% точкових акцентів — `--c-green`, `--c-bg-noir`

### 4.2.6 Scroll-reveal анімація footer

При в'їзді footer у viewport:

1. **0ms** — лицьова частина починає малювати будівлю (path-drawing 1500ms).
2. **300ms** — watermark «1 5 7» staggered fade-in (200ms кожна цифра).
3. **800ms** — печатка «EST. 1957» stamp-drop.
4. **1000ms** — seam перфорація з'являється (clip-path expand, 280ms).
5. **1100ms** — лист починає друкуватись слово за словом (Caveat words reveal, 60ms кожне, total ~3.5s).
6. **1500ms** — марка наклеюється з cinematic effect.
7. **2100ms** — штамп «АРХІВ» падає.
8. **2400ms** — штамп «ПЕРЕДАНО» падає поверх.
9. **2600ms** — адреси з'являються (translateY 16 → 0, opacity 0 → 1).

11. **3200ms** — копірайт-смуга slide up (translateY 80 → 0).

**Анімація на scroll-into-view:**
1. **Photo zoom**: `scale(1.06) → 1`, 1500ms `--ease-paper`. Дає «cinematic Ken Burns» effect.
2. **Цитата:** друкується слово за словом — кожне слово blur(8px) → 0 + opacity 0 → 1, stagger 80ms.
3. **Editorial label:** translateX(-20) → 0 + opacity 0 → 1, 480ms.
4. **Caption:** opacity 0 → 1, 600ms, delay 800ms.

**Hover-парадигма:**
- На бажання дизайнера — паралакс при mouse-move: photo зміщується на ±10px по обох осях зі швидкістю 60ms-trail.
- АБО навпаки — статика, абсолютна тиша. Я **рекомендую тишу** — це робить блок медитативним.

### Альтернативні цитати на вибір (на випадок A/B):

1. «Тут немає глядачів. Тільки автори.» ← **рекомендую**
2. «Школа не вчить творчості. Вона дає мовчання.»
3. «З 1957 року — той самий ритуал: інструмент і час.»
4. «Кожна робота — спершу мовчання. Потім — крик.»

### Mobile варіант

- Висота 480px.
- Cropping photo сильніший — фокус на одному обличчі/руках замість широкого плану.
- Цитата 48px, 3 рядки.
- Editorial label і caption переміщуються вище/нижче відповідно.
- Відступи 24px замість 64.

### Чому цей блок працює для cohesion

- Чорно-біле + бордовий duotone — той самий treatment, що в Manifesto (Block 4) → візуальний rhyme.
- Editorial label `▌` — повторює систему всієї сторінки.
- Печатка в кутку — повторюваний motif з 5+ випадків на сторінці.
- Рукописна нотатка — система маргіналій збережена.
- Цитата у Fraunces italic 96px — найбільший контраст з щоденним тоном, але ритмово працює як мега-pull-quote magazine.
- Photo не як «hero», а як **«центральний розворот»** — це чітка editorial-логіка, не e-commerce.

- Hover: bg burgundy, стрілка white.

**Пагінаційна лінія:**

- Manrope 13px muted.
- Активна — burgundy + перед цифрою `●`.
- Не клікабельна (тільки індикатор).

1. Мотузка «натягується» — SVG path stroke-dashoffset 100% → 0, 1200ms.
2. Кожен полароїд stagger-падає згори: translateY(-40) → 0 + rotate(rand) → final, stagger 120ms.

**Mobile:**
- Перспектива і сложна композиція спрощуються.
- Мотузка горизонтальна, але видно тільки 3 полароїди (lateral 80vw кожен).

- Музейний підпис під полароїдом, не справа.

| Performance hit (perspective + blur) | Lighter, simpler 2D transforms |

Перед прийняттям дизайну будь-якої сторінки — перевір:

### Кольори
- [ ] Background — `--c-bg` (#fafaf7) на 60%+ поверхні.
- [ ] Чисто білий (#ffffff) — ТІЛЬКИ в картках.
- [ ] Бордовий — primary акцент (заголовки, ціна, печатки).
- [ ] Зелений — secondary (навігація, success, hand-notes).
- [ ] Жовтий (#fef3c7) — стікери та game block.

### Paper artifacts
- [ ] Хоча б 1 печатка на сторінці (на головній — мін. 5).
- [ ] Хоча б 1 рукописна нотатка Caveat.
- [ ] Хоча б 1 нахилений елемент (polaroid / sticker).
- [ ] Editorial label з ▌ як мінімум один раз на сторінку.

### Anti-patterns
- [ ] Жодних purple/pink градієнтів.
- [ ] Жодного Inter/Roboto.
- [ ] Жодних емодзі в UI.
- [ ] Жодних generic Lottie animations.
- [ ] Жодного floating chat bubble.

### Photo treatment
- [ ] Усі ч/б фото — з бордовим duotone.
- [ ] Усі polaroids — біла рамка 12+36px.
- [ ] Photo overlay секції — vignette + bottom gradient для читабельності.

### Cohesion
- [ ] Чи виглядає це як «частина 47-го випуску архіву»?
- [ ] Чи можна знайти це у магазині 2030 — і одночасно у журналі 1957?
- [ ] Чи є хоч один patterns, що повторюється з головної?

# РОЗДІЛ XIX. ФОТОГРАФІЧНИЙ ГАЙДЛАЙН ТА ПРОМТИ

> **Ключова ідея:** Фотографії — це не реклама, це **документальне свідчення**. Вони мають виглядати так, ніби їх зняв талановитий шкільний фотограф на плівку в 1970-х або 2026-му.

## 19.1 Візуальний стиль: Archive-Docu
- **Колір:** Чорно-білі фото з легким теплим тонуванням (burgundy duotone) для блоків-інтерлюдій. Кольорові фото (для товарів) — приглушені, природні, з м'яким контрастом.
- **Світло:** Тільки природне віконне світло. Глибокі тіні, яскраві відблиски на пилу чи папері.
- **Текстура:** Легке зерно (film grain), ніякої «цифрової ідеальності».
- **Композиція:** Нецентрована, динамічна, «спійманий момент».

## 19.2 Категорії зображень

1.  **«АТМОСФЕРА (The Space)»:** Коридори ліцею, сонячні промені в класі мистецтв, порожній спортзал, сходи.
2.  **«ПРОЦЕС (The Craft)»:** Руки в глині, розлита акварель, заточені олівці, стружка від дерева, забризканий фартух.
3.  **«АВТОРИ (The Creators)»:** Непостановочні портрети. Учень замислено дивиться у вікно або зосереджений на малюнку. Ніяких посмішок у камеру.
4.  **«ОБОЛОНЬ (The Context)»:** Вид з вікна школи на Оболонські будинки, двір ліцею, вивіска.
